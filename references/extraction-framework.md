# Extraction Framework

Each source card should extract patterns useful for building a real Skill.

Source cards are also search briefs. Before collection, define which card types are needed and search for sources that can fill them. After collection, use the same card fields to extract evidence.

## Source-Card Search Fields

Before using anysearch or manual search, classify each query by intended card type:

- `official_methodology`
- `diagnostic_question`
- `tool_template`
- `case_practice`
- `video_transcript`
- `external_boundary`
- `china_localization`

Every candidate source should preserve this intent where possible. If search volume is weak, expand the empty or thin card types first instead of only adding generic company keywords.

## Extractable Fields

- HR / organization methodology
- diagnostic question
- tool / template
- management action
- applicable scenario
- limitation
- China localization implication
- evidence level
- source confidence

## Research Merge Rules

Assign source cards to research files by source type and tags:

- `01-official-methodology.md`: official reports, official articles, white papers, official methodology pages
- `02-video-transcripts.md`: videos, webinars, podcasts, transcripts
- `03-case-practices.md`: case studies, customer stories, practice cases
- `04-tools-and-frameworks.md`: tools, frameworks, templates, maturity models, diagnostic tables
- `05-external-views.md`: media, third-party analysis, academic critique, user feedback
- `06-timeline.md`: latest reports, recent articles, time-sensitive updates
- `07-china-localization.md`: Chinese sources, China cases, localization issues

## Final Skill Extraction

The generated Skill must include:

- role and boundary
- applicable scenarios
- core mental models
- diagnostic workflow
- evidence checklist
- tools and deliverables
- anti-patterns
- localization guidance
- source and uncertainty boundary
