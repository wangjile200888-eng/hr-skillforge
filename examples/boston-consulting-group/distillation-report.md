## 3. Method Principles / 方法原则

### 3.1 Advantage Starts With Capabilities / 竞争优势始于能力

BCG 的人才与技能相关公开资料反复把人才、技能、战略性人力规划和人员发展连接到企业竞争优势。蒸馏后的可用原则是：业务战略必须先转化成能力议题，再由能力议题推导关键岗位、技能缺口和人才动作。

Deliverables:

- Strategy-to-capability map / 战略到能力地图
- Critical-role heatmap / 关键岗位热力图
- Skill-gap and workforce gap table / 技能与人力缺口表
- Capability-building roadmap / 能力建设路线图

### 3.2 Performance Depends On Context / 绩效取决于组织情境

BCG 组织设计公开资料强调，组织绩效不仅由结构决定，也由员工如何行动、互动和决策决定。蒸馏后的可用原则是：组织设计必须同时处理结构、行为、决策权、流程、系统、协作接口和激励。

Deliverables:

- Organization-context diagnosis / 组织情境诊断
- Decision-rights redesign / 决策权重设
- Collaboration-interface map / 协作接口图
- Behavior-change plan / 行为改变计划

### 3.3 Distributed And Agile Models Need Explicit Design / 分布式与敏捷模式需要显性设计

BCG 分布式组织资料涉及平台组织、敏捷姿态、多学科团队、职能专家部署、共享服务、并购整合与拆分场景。蒸馏后的可用原则是：敏捷不是口号，而是任务、团队边界、专家配置、治理节奏和升级路径的组合设计。

Deliverables:

- Agile operating-model blueprint / 敏捷运营模式蓝图
- Platform capability map / 平台能力地图
- Team charter / 团队章程
- Governance and escalation matrix / 治理与升级矩阵

### 3.4 Leadership Development Must Happen In The Work / 领导力发展必须嵌入真实工作

BCG 领导力发展资料批评脱离场景的一次性培训，强调领导力发展要回应转型与业务场景。蒸馏后的可用原则是：领导力不是课程产品，而是围绕真实挑战、反馈、同伴机制和里程碑形成的行为改变系统。

Deliverables:

- Leadership requirement model / 领导力要求模型
- Leader behavior scorecard / 领导行为计分卡
- Action-learning journey / 行动学习旅程
- Leadership cadence and coaching plan / 领导力节奏与教练计划

### 3.5 AI And Digital HR Should Redesign Work / AI 与数字化 HR 应重塑工作

BCG 数字化 HR、BCG X、AI/GenAI 相关公开材料反复连接数字能力、人类能力与转型优势。蒸馏后的可用原则是：数字化 HR 的核心不是系统上线，而是工作、角色、技能、数据流、员工体验和 HR 服务共同重设。

Deliverables:

- AI work-redesign matrix / AI 工作再设计矩阵
- Digital HR roadmap / 数字化 HR 路线图
- People-data model / 人力数据模型
- HR service journey / HR 服务旅程
- Adoption dashboard / 采纳仪表盘

## 4. Practical Use Protocol / 实战调用协议

When using this Skill to solve a user's problem:

1. Start from the business advantage or transformation goal.
2. Translate the goal into organization capabilities, roles, skills, leadership behaviors, and work systems.
3. Diagnose the current context: decision rights, collaboration interfaces, incentives, data, routines, and governance.
4. Select intervention patterns that match BCG-style logic.
5. Produce concrete deliverables with tables, owners, metrics, timeline, and assumptions.
6. State evidence boundaries and ask for missing user data only when needed.

使用本 Skill 时，不应只讲 BCG 方法论，而要把方法论延伸到具体交付物和具体交付结果：例如组织诊断表、战略到能力地图、关键岗位与技能缺口表、敏捷运营模式蓝图、领导力行动学习计划、AI 工作再设计矩阵、转型采纳仪表盘。
