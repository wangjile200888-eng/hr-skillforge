## Core McKinsey-Style Logic / 麦肯锡风格核心逻辑

Use this five-layer logic before designing any HR solution.  
设计任何 HR 方案前，先使用五层逻辑。

```text
1. Value at stake / 价值所在
   What business outcome is constrained by people, structure, leadership, or culture?
   哪个业务结果正在被人才、组织结构、领导力或文化限制？

2. Strategic role of talent and organization / 人才与组织的战略角色
   Which roles, capabilities, operating-model choices, or leadership behaviors most affect that value?
   哪些岗位、能力、运营模式选择或领导行为最影响这部分价值？

3. Fact base and issue tree / 事实基础与议题树
   What evidence explains the gap: performance, org health, skills, roles, incentives, process, analytics, or change adoption?
   哪些证据解释差距：绩效、组织健康、技能、岗位、激励、流程、数据分析或变革采纳？

4. Intervention system / 干预系统
   Which combined interventions change outcomes: organization design, talent allocation, performance management, rewards, leadership, culture, HR operating model, people analytics?
   哪些组合动作能改变结果：组织设计、人才配置、绩效管理、薪酬激励、领导力、文化、HR 运营模式、People Analytics？

5. Implementation engine / 落地引擎
   How will leaders sequence action, build capability, track adoption, manage resistance, and sustain impact?
   管理层如何排序行动、建设能力、跟踪采纳、管理阻力并持续产生影响？
```

## Publicly Supported Method Families / 公开资料支持的方法族

### 1. People & Organizational Performance / 人才与组织绩效

Use this as the master lens. Public McKinsey material frames sustained performance as the result of aligning people, structures, systems, skills, leadership, culture, and change. Do not diagnose HR as an isolated function; connect it to strategy and enterprise performance.  
将其作为总镜头。麦肯锡公开资料将持续绩效视为人才、结构、系统、技能、领导力、文化与变革协同的结果。不要把 HR 当成孤立职能诊断，必须连接到战略和企业绩效。

Typical applications / 典型场景：

- Enterprise organization diagnosis / 企业组织诊断
- People strategy linked to business strategy / 与业务战略挂钩的人才战略
- Leadership and culture transformation / 领导力与文化转型
- Talent and skill-system redesign / 人才与技能体系重塑
- HR function transformation / HR 职能转型
- Performance and organizational health improvement / 绩效与组织健康提升

### 2. Talent To Value / 人才到价值

Use when the problem is talent allocation, critical roles, workforce scarcity, or strategic capability gaps.  
当问题涉及人才配置、关键岗位、劳动力短缺或战略能力缺口时使用。

Working logic / 工作逻辑：

```text
Identify value pools / 识别价值池
-> identify roles that disproportionately create or protect value / 识别对价值创造或价值保护影响不成比例的岗位
-> diagnose capability, capacity, and motivation gaps / 诊断能力、容量和动机缺口
-> match top talent to critical work / 将优秀人才匹配到关键工作
-> redesign talent processes around value, not hierarchy / 围绕价值而非层级重设人才流程
-> track business impact and retention / 跟踪业务影响与留任效果
```

Questions to ask / 关键问题：

- Which roles or teams create disproportionate value? / 哪些岗位或团队创造了不成比例的价值？
- Are top people allocated to the highest-value work? / 最优秀的人是否被配置到最高价值工作上？
- Which skill pools are scarce, strategic, or at risk? / 哪些技能池稀缺、战略性强或存在风险？
- What talent processes slow redeployment or internal mobility? / 哪些人才流程拖慢了重新配置或内部流动？
- Does HR act as a business-value driver or an administrative function? / HR 是业务价值驱动者，还是行政支持职能？

### 3. Organization Design And Organize To Value / 组织设计与围绕价值组织

Use when strategy is clear but execution is slow, accountability is blurry, spans/layers are inefficient, or the operating model no longer fits the business.  
当战略清楚但执行缓慢、责任模糊、管理跨度/层级低效，或运营模式不再适配业务时使用。

Working logic / 工作逻辑：

```text
Strategic aspiration / 战略愿景
-> value map / 价值地图
-> operating-model fingerprint / 运营模式指纹
-> decision rights and accountabilities / 决策权与责任
-> structure, roles, governance, processes, talent, culture / 结构、岗位、治理、流程、人才、文化
-> transition roadmap / 过渡路线图
```

Diagnostic focus / 诊断重点：

- Where is value created and where does work actually happen? / 价值在哪里创造，工作实际在哪里发生？
- Which decisions are slow, duplicated, or escalated too often? / 哪些决策慢、重复或过度上升？
- Which interfaces create friction between business, product, function, geography, and shared services? / 哪些接口在业务、产品、职能、区域和共享服务之间制造摩擦？
- Are roles designed around value and accountability or around legacy reporting lines? / 岗位是围绕价值与责任设计，还是围绕历史汇报线设计？
- What must change in governance, performance metrics, and leadership routines for the new model to work? / 新模式要生效，治理、绩效指标和领导例会机制必须如何改变？

### 4. Performance Management That Puts People First / 以人为先的绩效管理

Use when performance reviews, goals, ratings, feedback, rewards, or development systems are demotivating or disconnected from strategy.  
当绩效评估、目标、评级、反馈、激励或发展体系削弱动机，或与战略脱节时使用。

Working logic / 工作逻辑：

```text
Strategy and performance ambition / 战略与绩效目标
-> goal architecture / 目标架构
-> performance conversations and feedback cadence / 绩效对话与反馈节奏
-> differentiation and consequence logic / 区分度与结果机制
-> development and career paths / 发展与职业路径
-> reward and recognition mix / 奖励与认可组合
-> manager capability / 管理者能力
```

Design questions / 设计问题：

- What should the system optimize: individual accountability, team outcomes, capability development, value creation, or behavior change? / 系统要优化什么：个人责任、团队结果、能力发展、价值创造还是行为改变？
- Is goal setting linked to strategic priorities? / 目标设定是否连接战略优先级？
- Is feedback frequent enough to change behavior? / 反馈是否足够频繁以改变行为？
- Are managers able to coach, differentiate, and have hard conversations? / 管理者是否能辅导、区分绩效并进行艰难对话？
- Are monetary and non-monetary rewards balanced? / 金钱与非金钱激励是否平衡？
- Does the system motivate high performers without creating fear, bureaucracy, or gaming? / 系统是否激励高绩效者，同时避免恐惧、官僚和博弈？

### 5. Leadership And Organizational Health / 领导力与组织健康

Use when the user asks about leadership development, culture, transformation, or long-term performance.  
当用户询问领导力发展、文化、转型或长期绩效时使用。

Working logic / 工作逻辑：

```text
Business context / 业务背景
-> leadership behaviors required by strategy / 战略要求的领导行为
-> organizational health and culture signals / 组织健康与文化信号
-> leadership capability gaps / 领导力能力缺口
-> top-team and manager routines / 高管团队与管理者例行机制
-> leadership academy / capability-building system / 领导力学院或能力建设体系
-> measurement and reinforcement / 衡量与强化
```

Typical leadership lenses / 典型领导力视角：

- Leadership strategy tied to business strategy / 与业务战略绑定的领导力战略
- Team effectiveness / 团队有效性
- Capability building at scale / 规模化能力建设
- Mindsets and behaviors that sustain transformation / 支撑转型的心智与行为
- Organizational health as a leading indicator of performance / 将组织健康作为绩效的领先指标

### 6. Change Management And Transformation / 变革管理与转型

Use when a technically good HR or organization design fails because behavior, adoption, incentives, or leadership alignment are weak.  
当技术上正确的 HR 或组织设计因为行为、采纳、激励或领导层对齐不足而失败时使用。

Working logic / 工作逻辑：

```text
Transformation aspiration / 转型愿景
-> stakeholder map / 利益相关方地图
-> behavior shifts / 行为转变
-> influence model / reinforcement system / 影响模型与强化系统
-> capability building / 能力建设
-> adoption metrics / 采纳指标
-> execution cadence / 执行节奏
```

Diagnose / 诊断：

- Who must behave differently? / 谁必须改变行为？
- What incentives or rituals reinforce old behavior? / 哪些激励或仪式强化旧行为？
- Which leaders are aligned, passive, or resistant? / 哪些领导者支持、被动或抵触？
- What capability must be built before the new model can work? / 新模式生效前必须建设什么能力？
- What adoption metrics show real behavior change rather than activity? / 哪些采纳指标能证明真实行为改变，而不只是活动发生？

### 7. People Analytics / 人才数据分析

Use when the organization has data but lacks insight, or when leaders rely on anecdotes to make people decisions.  
当组织有数据但缺洞察，或领导者依赖轶事经验做人才决策时使用。

Working logic / 工作逻辑：

```text
Business question / 业务问题
-> people hypothesis / 人才假设
-> data availability and quality / 数据可得性与质量
-> analysis and segmentation / 分析与分群
-> insight validation / 洞察验证
-> action design / 行动设计
-> impact tracking / 影响跟踪
```

Use people analytics for / People Analytics 可用于：

- Workforce planning / 人力规划
- Attrition and retention / 流失与留任
- Talent allocation / 人才配置
- Skills and capability gaps / 技能与能力缺口
- Performance drivers / 绩效驱动因素
- Organizational network and collaboration patterns / 组织网络与协作模式
- HR process effectiveness / HR 流程有效性

### 8. HR Operating Model / HR 运营模式

Use when HR is seen as administrative, slow, fragmented, or not close enough to business value.  
当 HR 被认为行政化、缓慢、碎片化或不够贴近业务价值时使用。

Working logic / 工作逻辑：

```text
Business strategy / 业务战略
-> HR value proposition / HR 价值主张
-> work segmentation / 工作分层
-> HRBP / COE / shared services / digital roles / HRBP、COE、共享服务与数字化角色
-> governance and service model / 治理与服务模式
-> people analytics and productized HR services / People Analytics 与产品化 HR 服务
-> capability and change roadmap / 能力与变革路线图
```

Questions / 关键问题：

- Which HR work creates strategic value and which should be standardized or automated? / 哪些 HR 工作创造战略价值，哪些应标准化或自动化？
- Are HRBPs acting as strategic advisers or issue escalators? / HRBP 是战略顾问，还是问题上报通道？
- Are COEs designing usable products or policy documents? / COE 设计的是可用产品，还是政策文件？
- Is shared services measured on experience and resolution quality? / 共享服务是否按体验和解决质量衡量？
- Does HR use data and product thinking to improve adoption? / HR 是否用数据和产品思维提高采纳？

## Diagnostic Question Sets / 诊断问题组

### A. Enterprise People Strategy / 企业人才战略

- What is the business strategy, and which people/organization constraints can prevent it? / 业务战略是什么，哪些人才或组织约束会阻碍它？
- Which value pools depend most on talent, leadership, culture, or operating-model choices? / 哪些价值池最依赖人才、领导力、文化或运营模式选择？
- Which capabilities must be built, bought, borrowed, or automated? / 哪些能力需要自建、购买、借用或自动化？

### B. Talent Allocation / 人才配置

- Which roles have disproportionate value or risk? / 哪些岗位具有不成比例的价值或风险？
- Where are your best people currently deployed? / 最优秀的人现在部署在哪里？
- What prevents dynamic internal mobility? / 什么阻碍了动态内部流动？
- Which skills will become obsolete or scarce in the next 12-36 months? / 未来 12-36 个月哪些技能会过时或稀缺？

### C. Organization Design / 组织设计

- Does the structure reflect strategy or legacy power? / 组织结构反映的是战略，还是历史权力？
- Where are decision rights unclear? / 决策权在哪里不清楚？
- Which spans, layers, interfaces, and governance forums slow execution? / 哪些跨度、层级、接口和治理会议拖慢执行？
- What accountabilities must be explicit before changing boxes on an org chart? / 调整组织图之前，哪些责任必须先明确？

### D. Operating Model / 运营模式

- What work belongs in business units, functions, platforms, shared services, or ecosystems? / 哪些工作属于业务单元、职能、平台、共享服务或生态？
- Which processes require global consistency versus local autonomy? / 哪些流程需要全球一致，哪些需要本地自主？
- Where does the current model create handoff friction? / 当前模式在哪里制造交接摩擦？
- Which metrics prove the model is working? / 哪些指标能证明模式有效？

### E. Performance Management / 绩效管理

- Are goals clear, strategic, and adaptable? / 目标是否清晰、战略相关且可调整？
- Do reviews improve performance or just document it? / 评估是在提升绩效，还是只是记录绩效？
- Are rewards linked to contribution, behavior, capability, and retention? / 激励是否连接贡献、行为、能力和留任？
- Do managers have enough skill and time to coach? / 管理者是否有足够能力和时间辅导员工？

### F. Rewards And Incentives / 薪酬激励

- What behaviors do current rewards actually encourage? / 当前激励实际鼓励了什么行为？
- Are incentives aligned with value creation and risk control? / 激励是否与价值创造和风险控制一致？
- Are non-cash motivators being used deliberately? / 是否有意识地使用非现金激励？
- Where do rewards create internal inequity or short-termism? / 激励在哪里制造内部不公平或短期主义？

### G. Leadership / 领导力

- What leadership behaviors does the strategy require? / 战略需要什么领导行为？
- Which leader population matters most right now: executives, middle managers, frontline leaders, critical experts? / 当前最关键的领导群体是谁：高管、中层、一线主管还是关键专家？
- How will leadership development move from training events to embedded routines? / 领导力发展如何从培训活动变成嵌入式管理例行机制？

### H. Culture And Change / 文化与变革

- What behavior must change for the strategy to work? / 战略要生效，哪些行为必须改变？
- Which symbols, routines, incentives, and stories reinforce the old culture? / 哪些符号、例行机制、激励和故事强化旧文化？
- Which leaders can model the new behavior visibly? / 哪些领导者能可见地示范新行为？
- How will adoption be measured? / 如何衡量采纳？

### I. People Analytics / 人才数据分析

- What decision will improve if analytics improves? / 如果数据分析改善，哪个决策会变好？
- What is the hypothesis? / 假设是什么？
- What data is reliable enough? / 哪些数据足够可靠？
- How will insights become action rather than dashboards? / 洞察如何转化为行动，而不是仪表盘？

### J. China Localization / 中国本土化

- Which global concepts need translation into Chinese management language? / 哪些全球概念需要转化成中国管理语言？
- How do hierarchy, labor-market conditions, incentive norms, and regulatory constraints change implementation? / 层级、劳动力市场、激励习惯和监管约束如何改变落地方式？
- What must be simplified for Chinese executives and line managers to use repeatedly? / 哪些工具必须简化，才能被中国高管和一线经理反复使用？

## Answer Workflow / 回答工作流

```text
1. Classify the problem domain / 判断问题域
   strategy, organization design, talent, performance, rewards, leadership, HR transformation, culture/change, analytics
   战略、组织设计、人才、绩效、薪酬激励、领导力、HR 转型、文化/变革、数据分析

2. Build an issue tree / 构建议题树
   split the problem into value, facts, root causes, options, risks, and implementation
   将问题拆成价值、事实、根因、选项、风险和落地

3. State hypotheses / 提出假设
   make 2-4 testable hypotheses before recommending solutions
   推荐方案前先提出 2-4 个可验证假设

4. Recall evidence / 回看证据
   official methodology -> reports/PDFs -> tools/articles -> cases -> podcast/video candidates -> external boundary
   官方方法论 -> 报告/PDF -> 工具/文章 -> 案例 -> 播客/视频候选 -> 外部边界

5. Design the solution system / 设计解决方案系统
   combine structure, process, people, incentives, data, leadership, and change
   组合结构、流程、人才、激励、数据、领导力和变革

6. Translate to action / 转化为行动
   roadmap, owners, cadence, metrics, decision forums, communication, capability building
   路线图、负责人、节奏、指标、决策会议、沟通和能力建设

7. Mark confidence / 标注置信度
   high confidence, medium confidence, candidate inference, or needs new research
   高置信、中置信、候选推断或需要新研究
```

## Confidence Labels / 置信度标签

- **High confidence / 高置信：** repeated official evidence or major McKinsey reports support the point. / 多个官方证据或重要麦肯锡报告支持。
- **Medium confidence / 中置信：** supported by official pages plus source cards, but not deeply evidenced. / 有官方页面和证据卡支持，但深度证据不足。
- **Candidate inference / 候选推断：** logical synthesis from public evidence, not a named McKinsey model. / 基于公开证据的逻辑综合，不是麦肯锡命名模型。
- **Needs verification / 需要验证：** current facts, platform-restricted video wording, company-specific data, laws, pay data, or private project claims. / 最新事实、受限视频原话、具体公司数据、法律、薪酬数据或私人项目说法需要验证。

## Anti-Patterns / 反模式

- Starting with HR initiatives before defining business value at stake. / 未定义业务价值就启动 HR 项目。
- Treating organization design as only structure boxes. / 把组织设计等同于画组织架构图。
- Treating performance management as annual rating mechanics only. / 把绩效管理等同于年度评级。
- Building leadership programs disconnected from strategy and manager routines. / 领导力项目与战略和管理者例行机制脱节。
- Using people analytics as dashboards without decisions and interventions. / People Analytics 只做仪表盘，不连接决策和干预。
- Confusing employee engagement activity with organizational health and performance. / 把员工活动误认为组织健康和绩效。
- Copying global methods into China without adapting language, governance, labor rules, and managerial habits. / 直接照搬全球方法，不适配中国语言、治理、劳动规则和管理习惯。
- Presenting public McKinsey synthesis as proprietary McKinsey method. / 将公开资料综合包装成麦肯锡专有方法。

## China Localization / 中国本土化

When localizing for China, translate concepts into management language that Chinese executives and line managers can use repeatedly.  
在中国落地时，要把概念转化为中国高管和一线经理能反复使用的管理语言。

- Translate `People & Organizational Performance` into business language: 人效、组织效能、人才供给、干部能力、组织健康、变革落地.
- Link talent and organization topics to growth quality, productivity, managerial leverage, and execution discipline. / 将人才与组织议题连接到增长质量、生产率、管理杠杆和执行纪律。
- Consider labor-market constraints, pay transparency norms, hierarchy, regional differences, and compliance. / 考虑劳动力市场约束、薪酬透明习惯、层级文化、区域差异和合规。
- Make tools manager-friendly: one-page issue tree, diagnostic questions, role/accountability map, KPI tree, cadence table. / 工具要经理友好：一页纸议题树、诊断问题、岗位/责任图、KPI 树、节奏表。
- Avoid over-academic language; Chinese executives often need a direct connection to revenue, margin, growth, risk, and management action. / 避免过度学术化；中国高管通常需要直接看到收入、利润、增长、风险和管理动作的连接。
